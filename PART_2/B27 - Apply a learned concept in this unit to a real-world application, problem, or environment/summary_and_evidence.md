I applied the cybersecurity concept of password security to a real-world network environment by relating it to how devices are identified and accessed using Internet Protocol addresses and Media Access Control addresses. In a home, university, or workplace network, every device has a Media Access Control address and is usually assigned an Internet Protocol address so it can communicate on the network. However, even though these addresses help identify devices, they do not by themselves prove that a user is authorised. This means that if account passwords are weak or reused, an attacker who gains access to a device or network can still misuse services tied to that device, regardless of its Internet Protocol or Media Access Control identity.

A practical example is a student laptop connected to a university or home Wi-Fi network. The laptop has its own Media Access Control address and receives an Internet Protocol address from the router. If the student uses weak or reused passwords for important accounts, an attacker who compromises the account can still access email, cloud storage, or online services even though the device itself is known on the network. This shows that device identity on a network is not enough on its own; strong password security is still needed to protect the accounts being accessed from that device.

The practical application of this concept is in real network and account protection. Internet Protocol and Media Access Control addresses help identify where traffic is coming from and what device is involved, while password security protects the actual user accounts and services. This activity showed that password security works together with network identification, but cannot be replaced by it. Strong, unique passwords are still necessary to reduce the risk of unauthorised access in real environments.
