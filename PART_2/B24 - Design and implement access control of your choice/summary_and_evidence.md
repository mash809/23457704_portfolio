I designed and implemented a simple role-based access control system. Role-based access control is a method where users are given permissions based on their role, rather than giving every user the same access. This helps protect systems by making sure people can only access the resources and actions that are appropriate for them.

In my implementation, I created different user roles such as student, lecturer, and administrator. Each role had different permissions. For example, a student could view their own information, a lecturer could manage academic records, and an administrator had full control over the system. When a user selected a role, the program checked what actions were allowed and displayed either access granted or access denied.

This implementation showed how access control can reduce security risks by limiting what each user can do. It prevents unauthorised users from viewing or changing sensitive information and follows the principle of least privilege, which means users should only receive the minimum access they need. However, the system is still simple and would need stronger authentication and more detailed permission management in a real-world environment.

Overall, this activity helped me understand that access control is one of the most important security concepts because it protects systems by ensuring that only the right users can perform the right actions.
