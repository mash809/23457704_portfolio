I tested a basic intrusion detection style setup using Windows Defender Firewall logging. Because the firewall log is stored in a protected system folder, administrator access was needed to review it. After enabling logging and generating network traffic, I attempted to inspect the recorded entries to understand how Windows logs blocked or monitored connections.

This activity showed that monitoring tools can provide useful visibility into system and network activity, but they also depend on correct permissions and configuration to be useful. It also showed that built-in logging is more limited than a dedicated intrusion detection system, because it records events but does not provide deeper threat analysis. Overall, the test demonstrated that detection tools are helpful for visibility, but stronger security monitoring needs more advanced tools and setup.
